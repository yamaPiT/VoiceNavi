# ユニットテスト生成ワークフロー (ADAプロセス準拠)

以下の手順でプロジェクトのユニットテスト（自動テスト）を自律的に計画・生成・実行してください。

1. **テスト計画の策定 (Plan)**
   - 変更されたファイル、またはユーザが指定したファイルとその中の主要なメソッド・コンポーネントを特定し、「何をどうテストするか」の計画を立ててマサに提案（レビュー依頼）します。
2. **テストコードの生成 (Implement)**
   - 提案が承認されたら、指定された対象に対して網羅的なユニットテストを生成します。
   - `vitest` 等、プロジェクトで採用されているテストフレームワークの構文を使用してください。
3. **命名規則の遵守**
   - 生成するユニットテストのファイル名は、元のファイル名と同じベース名で、`*.test.ts` あるいは `test_*.ts` といったプロジェクト固有の規則に従って保存してください。
4. **自律的検証と修復 (Verify & Auto-Heal)**
   - テストを生成した後は必ず実行し、エラーが発生した場合はAI自身がエラーログを解析し、テストやコードを自己修正（セルフヒーリング）してすべてがPassするまで対応します。
