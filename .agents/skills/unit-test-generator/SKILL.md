# ユニットテスト自律生成・修復 (Unit Test Generator)

本スキルは、実装フェーズ（Phase 4）において、ユニットテストを人間の関与なしに裏側で完結させるための物理的な自動実行ループの手順です。

## 実行手順 (Mechanical How)

1. **内部計画 (No Human Review)**:
   - 対象ファイルに対するテスト計画をAIコンテキスト内部だけで立案します。
   - **【厳守】この計画をProduct Owner（人間）に提案・レビュー依頼してはいけません。即座に次へ進みます。**
2. **テストコードの生成と配置**:
   - `vitest` 等のプロジェクト固有のテストフレームワーク構文に従い、テストファイルを生成します（例: `*.test.ts`）。
3. **完全自律デバッグループ**:
   - ターミナルコマンド等でテストを実行します。
   - エラーが発生した場合は**人間に報告せず**、自らエラーログを読み取り、コードまたはテストを修正して再実行します。
   - すべてのテストが「グリーン（成功）」になるまで、このループを自律的に繰り返します。
4. **最終完了報告**:
   - オールグリーンを達成した後のみ、上位のWorkflowに対して「実装と単体テストが完了した」という結果のみを報告してください。
