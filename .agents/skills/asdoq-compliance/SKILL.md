# ASDoQ文書品質モデルへの準拠 (ASDoQ Compliance)

すべての成果物（要求仕様書、設計書、実装計画などのドキュメント）において、ASDoQ システム開発文書品質モデル の以下の特性を担保すること。
※ASDoQモデルの詳細な定義と基準については、プロジェクト内のパス `docs/asdoq_model_markdown.md` を読み込んで参照すること。

1. **目的適合性**: 要求仕様を実現するために必要な情報が不足なく記述・実装されていること。
2. **保守性**: 将来の仕様変更に備えモジュール間の結合度を低く保ち、論理的な階層構造を持たせること。
3. **整合性**: 定義された用語のみを使用し、ドキュメント間およびコード間で矛盾がないこと。
4. **理解容易性**: 曖昧な表現を避け具体的に記述し、複雑なロジックには処理フローや状態遷移のコメントを付記すること。
5. **可読性**: インデントや改行を適切に使い、簡潔な箇条書きを活用すること。
6. **規範適合性**: プロジェクトルールおよび指定されたコーディング規約を遵守すること。
