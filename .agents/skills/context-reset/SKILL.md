# コンテキスト・リセット (Context Reset)

## 発動条件 (Trigger)
- `DADA-Process.md` の各工程における「承認ゲート」を通過した直後。
- Product Ownerから「承認します」「次へ進んで」等の許可が出た直後。

## 実行手順 (Mechanical How)
1. **サマリーの保存**: `docs/process/last_phase_summary.md` に以下を書き出す。
   - **完了フェーズ**: 完了したPhase番号と名称（例: Phase 1 - 要求定義）
   - **次フェーズ**: 次に着手すべきPhase番号と名称
   - **承認済みドキュメント一覧**: 各ドキュメントの状態（承認済/未作成）を更新
   - **引き継ぎ事項**: 以下を必ず含める
     - Product Ownerから受けた特別な指示や優先事項
     - 未解決のTBD項目や技術的リスク
     - 次フェーズで特に注意すべきポイント
     - スケール判定結果（Hotfix/Minor/Major）とその理由
2. **情報の破棄宣告**: 「未確定な議論をメモリから消去します」と宣言。
3. **Product Ownerへの依頼**: 「新しいチャットウィンドウで指示をください」と依頼。
