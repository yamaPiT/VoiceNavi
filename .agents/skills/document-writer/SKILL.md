# ドキュメント出力職人 (Document Writer)

本スキルは、各フェーズのRole（専門家）が作成・修正した開発ドキュメントを、現実のファイルシステムに安全かつ正確に反映させるための作業手順です。

## 実行手順 (Mechanical How)

1. **保存先の特定**: 
   - 開発成果文書（要求仕様書、設計書、テスト計画書など）は、必ず `docs/artifact/` ディレクトリ配下に保存してください。
2. **安全な上書きの徹底**:
   - 既存のファイルを更新する場合、いきなり全体を上書きせず、必ず変更前の内容と差分（Diff）を確認し、意図しない箇所の欠落を防いでください。
3. **フォーマットの維持**:
   - Markdownの表組み、Mermaid図のシンタックス、見出しの階層が崩れないように細心の注意を払って出力してください。
4. **完了の報告**:
   - ファイルの書き出し・更新が完了した事実と、保存したファイルパスのみを速やかに報告してください。（内容の解説は不要です）
