# SWP6 ソフトウェア総合テスト仕様書・報告書

## 1. テスト目的
本テストは、ソフトウェア要求仕様書（SW105）に追記された「フェーズ2: 10項目の機能改善・バグ修正」が正しく実装され、意図通りの動作であることを検証することを目的とする。

## 2. テスト環境
- **ブラウザ**: モダンWebブラウザ
- **バックエンド**: Node.js BFFサーバー ※**BFFサーバー(server.js) および Viteサーバー を必ず再起動**してから検証してください。

## 3. テストケース（フェーズ2 改善項目）

| テストID | テスト対象項目 | 操作・確認手順 | 期待される結果（Acceptance Criteria） | テスト結果 |
| :--- | :--- | :--- | :--- | :---: |
| T001 | (1) UI発話リストバグ | デモ中（待機中およびAI発話中）に、右側の「期待する発話リスト」をクリックする。 | AIが直ちに反応し、画面左側に発話内容が反映され、回答が返ってくること。 |  |
| T002 | (2) TTSの頭切れ解消 | AIの音声案内（システム発話を含む）を最初から最後まで聞く。 | 音声の冒頭が欠ける現象がなく、スムーズに発声が開始されること。 |  |
| T003 | (3) 音楽UI表示 | 発話リストから「サザンかけて」をクリックする。 | 画面上部ヘッダーの「時速表示（km/h）」の**左側**に、音符（🎵）と曲名が表示されること。 |  |
| T004 | (4) SSMLタグ除去 | 音声案内が行われた際の、画面左側のAIチャット吹き出しテキストを確認する。 | `<speak>` や `<break>` 等のSSMLタグ文字列がテキスト上に表示されていないこと。 |  |
| T005 | (5)(6) 座標修正 | 地図上の出発地点と、赤い最終目的地のピン位置を確認する。 | 出発が鎌倉駅前の通りであり、目的地が七里ヶ浜海岸駐車場の鎌倉側の位置に正しく設定されていること。 |  |
| T006 | (8-1) 250m視程ルール | スタート直後や直線途中のAIの自発的な案内内容を聞く。 | 遠く（250m以上）の目的地については「しばらく行くと〜」と言い、「前方に〜見えます」と発話しないこと。 |  |
| T007 | (8-2) 左右判定 | 海沿い等を走行中、周辺案内時の「右手」「左手」に言及する会話を聞く。 | 実際の進行方向に基づく物理的関係性（例: 由比ヶ浜を西へ進む際は海(南)が左手）と発話が一致していること。 |  |
| T008 | (9) なめりかわ交差点減速 | デモ走行中、「なめりかわ交差点（海沿いに出るT字路）」への接近と通過を観察する。 | 交差点に近づくと徐々に減速し、通過時に約**10km/h**まで落ち、その後再び50km/hに向け加速すること。 |  |
| T009 | (10) なめりかわ・発話制約 | デモ全体のAIの発話および画面テキストを確認する。 | 「滑川」が全て「なめりかわ」と発音・表示され、また「マミコが喜ぶ」系統のフレーズが全体で1回以下であること。 |  |

## 4. テストケース（フェーズ3 追加要望）

| テストID | テスト対象項目 | 操作・確認手順 | 期待される結果（Acceptance Criteria） | テスト結果 |
| :--- | :--- | :--- | :--- | :---: |
| T010 | (1) 煙たい・内気循環 | ドライバーが「前のトラックが煙たい」と発音する。 | AIが「内気循環にするね」と答え、画面中央に【車両制御】のシステムログが表示されること。 |  |
| T011 | (2) 発話例クリック無効 | 右側の「ドライバーの発話例」リストをクリックする。 | テキストがチャット欄に入力されず、AIも反応しないこと（単なる参考資料としての挙動）。 |  |
| T012 | (3) 目的地到着発話 | デモ走行を目的地（駐車場）まで完走させる。 | 停車後、AIが自動的に「目的地に到着しました」といった内容を（LLM経由で）発話すること。 |  |
| T013 | (4) Good Mellows制約 | Good Mellowsを通過するまで「お腹すいた」等と言わずに走行し、通過後に「お腹すいた」と言う。 | 通過前かつ空腹要望時のみ紹介され、通過後や要望がない時には紹介されないこと。 |  |
| T014 | (5) 自律発話のリミット | 同じ観光スポット（例：由比ヶ浜）の案内を複数回繰り返させる。 | 同じ内容の案内がデモ全体で最大2回までに抑えられていること。 |  |
| T015 | (6) リストの更新 | 画面右側のリストのタイトルと内容を確認する。 | タイトルが「ドライバーの発話例」となっており、指示された5つの例が表示されていること。 |  |
