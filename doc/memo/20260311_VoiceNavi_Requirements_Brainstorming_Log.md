# VoiceNavi ＆ ADAプロセス要件定義 壁打ち議事録 (2026/03/11)

## 1. プロジェクトの背景と目的（WHY）
*   **マサさんの原体験**: 1991年、デンソー基礎研究所にて「音声によるカーナビゲーション（画面を見ない安全なナビ）」を研究。恩師・健三さんのビジョン。
*   **当時の課題**: 音声認識の精度が低く、短い単語（応答語）での対話に限定されていたが、不景気により研究中断。
*   **今回の目的**: 生成AI（LLM）の進化により完全な自然言語対話が可能になった今、当時のビジョンが正しかったことを実証するため、VoiceNaviのデモンストレーションシステムを構築する。

## 2. 決定したシステムコンセプトと要件
*   **基本構成**: PCブラウザ上で動作するWebアプリケーション（画面に地図・テキスト、音声入出力を伴う）。
*   **コア機能**:
    1.  **ドライバーの過去の傾向 × 地理条件の掛け算**: 曖昧な発話から、LLMが意図と条件（口コミ評価など）を抽出してパーソナライズされた提案（休憩スポットなど）を行う。
    2.  **メンタルマップの形成支援**: 「右手の〇〇の手前」など、ドライバーが空間を頭の中に描きやすい相対的なナビゲーションを積極的に行う。
    3.  **コンシェルジュ・車両操作**: 「暑い」「花粉かな」といった自然言語から、安全上問題のない車両操作（エアコン・窓など）を擬似的に実行する。
    4.  **ガードレール（お断り）機能**: システムの物理的限界（車外の看板の読み取り等）や、交通違反（信号無視・速度超過）の要求に対しては丁寧に拒否する。
    5.  **デモシナリオの固定**: 再現性を高めるため、走行経路は「湘南海岸・鎌倉ドライブコース（国道134号線沿い）」を想定したシミュレーションとする。

## 3. Agentic Coding (自律AI開発) に向けたメタレベルの進化
マサさんの「AIエージェントによる並列開発・自律実行の強みを引き出す要求仕様書が必要」という深い洞察に基づき、ADAプロセスとSW105（要求仕様書）を「Agentic V字モデル」へとブラッシュアップした。

*   **Agentic Testability (テスト容易性の担保)**:
    テストエージェントが自動でテストコードを組めるよう、各シナリオに対して明確な「検証条件（Acceptance Criteria）」（例：JSONの出力フラグ期待値）を仕様に明記。
*   **Agentic Architecture (AI分散開発向け疎結合モジュール)**:
    複数AIが同時開発しても競合しないよう、「音声通信」「LLM対話」「地図UI」モジュールを完全に分離し、それぞれがモックを用いて単体テスト可能な設計を要求。
*   **Autonomous Feedback Loop (自律改善ループ)**:
    エラー（APIタイムアウトやパース失敗）時にプログラムが停止するのではなく、システム自身が繋ぎの音声を流しながらバックグラウンドで自己修復・リトライを行う異常系フォールバック要件を定義。
