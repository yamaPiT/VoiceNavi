# AIエージェントとの協調開発ガイド (Agent-In-The-Loop Development)

本ドキュメントは、「OSDVIスマートシナリオプロトタイプシステム (Smart Scene Simulator)」を題材として、**AIエージェント（Antigravity等）と開発者（人間）がいかに協調してソフトウェアを構築・保守していくか**のベストプラクティスをまとめた学習教材です。

---

## 1. はじめに：AIエージェントとの「ペアプログラミング」とは
これまでの開発では、エンジニアがすべてのコードをタイプし、人間がテストシナリオを考案して手動で実行していました。
AIエージェントを用いた開発では、人間（例：プロダクトオーナーのマサ）が**「要求（What）」**を提示し、AIエージェント（例：ハル）が**「実装（How）」**を自律的に担当します。

エージェントは単なるチャットボットではなく、プロジェクト全体のファイル群を読み書きし、ターミナルでコマンドを実行し、ブラウザを立ち上げてテストを行う「もう一人の優秀なエンジニア」として振る舞います。

---

## 2. 実践的なエージェント活用例（本プロジェクトでの事例）

### 事例A: 要件定義から実装への自律的サイクルの確立
人間が仕様の変更（例：「サンキューハザードの待機時間を変更してほしい」）を伝えると、AIエージェントは即座にコードを書き換えるのではなく、以下のステップを踏みます。
1. **影響範囲の調査**: 関連するソースコードや既存の仕様書（`SW105`, `SW205`など）を読み込みます。
2. **実装計画の策定**: `docs/Implementation_plan.md` 等を作成し、「どのファイルをどう書き換えるか」を人間に提案（レビュー依頼）します。
3. **安全な実装**: 人間の承認（Approve）を得た後、コードを書き換えます。
4. **自己完結型ドキュメントの更新**: コードの変更に合わせて、仕様書（上位ドキュメント）自体の記述も最新状態にアップデートし、システムの完全性を保ちます。

### 事例B: 網羅的なユニットテストの自動生成
「テストを書いて」という抽象的な指示から、AIエージェントは**プロジェクト全体を俯瞰し、必要なテストを網羅的に生成・実行**します。

#### 「特定のファイルのお試し」を超えたエージェントの動き
1. **対象ユニットの洗い出し**: `src/lib/` や `src/components/` 内から、ビジネスロジックや状態管理（例：`store.ts`）、UIコンポーネントなどの「テストが必要な部品（ユニット）」を自動抽出します。
2. **テスト計画の提示**: 「これらのファイルに対してVitestを用いたテストを構築しますが良いですか？」と人間に確認をとります。
3. **テストコードの大量生成**: 承認後、AIは高速で各ファイルに対応する `*.test.ts` を生成していきます。
4. **検証と修正（セルフヒーリング）**: 生成したテストを一括実行（`npx vitest run`）し、もしエラーが出た場合は、**AI自身がエラーログを読んで原因を解析し、コードやテストを自己修正**して再度実行します。
5. **完了報告**: 全てのケースが自動的にPassした（オールグリーンになった）時点で人間に報告します。

## 3. エージェントを上手く操る（プロンプトエンジニアリング）のコツ

AIエージェントから最大のパフォーマンスを引き出すためには、以下のポイントを意識して指示を出します。

1. **ワークフローやスキルの明示的利用**:
   * プロジェクトに用意された**「スラッシュコマンド (`/`)」**を効果的に使います。
   * 例: 単純にお願いするのではなく、`@[/generate-unit-tests] 全コンポーネントのテストを作成して` のようにコマンド（手順書）への参照を付けることで、AIはプロジェクト固有のルール（命名規則やフレームワーク）に従って高い精度で動作します。

2. **「何を（What）」と「どうやって（How）」の分離**:
   * 「このファイルのこの行の変数をAからBに変えて」と細かく指示するより、「雨量の判定基準を10%から15%に上げたい。関連システムへの影響を最小限にして実装して」と**目的を指示する**方が、AIはアーキテクチャ全体を考慮した最適な実装（テスト修正やドキュメント更新を含む）を提案できます。

3. **コンテキスト（背景情報）の共有**:
   * 今、自分がどういう目的で変更を行おうとしているのか（「新規機能追加なのか」「リファクタリングなのか」「教材化の準備なのか」）を伝えることで、AIは行動指針（どこまで広範囲に手を加えるべきか）を自己決定しやすくなります。

---

## 4. 総括
AIエージェントとの協調開発において、人間の主なロール（役割）は「コードの記述者」から、**「要件の定義者・システムの方向性の決定者（レビューア）」**へとシフトします。
AIの自律的な調査能力、実装力、検証（テスト）能力を信頼しつつ、重要な設計判断のフェーズで人間が手綱を握る（Approveする）ことが、現代のソフトウェア開発における最も効率的で強力なアプローチです。
