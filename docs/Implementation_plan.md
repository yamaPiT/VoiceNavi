# Implementation Plan: ドキュメントの自己完結化対応 (SW105, SW205)

## 1. 目的と範囲
本計画は、要求仕様書（SW105）およびソフトウェアアーキテクチャ設計書（SW205）において、他段階のドキュメントを参照させる「（※前回から変更なし）」などの記述を排除し、各文書単体で仕様が完全に理解できるよう自己完結させるための改訂手順を定義する。
※本件はドキュメントの改訂自体が目的であるため、ソースコード（機能ロジック）の変更は伴わない。

## 2. 実施手順 (Plan)
1. **既存仕様の明確化 (完了)**:
   * ソースコード（`Store.ts` など）および `README.md` から、省略されていた仕様（サンキューハザード、実行エンジン）の具体的内容を抽出。
2. **SW105（要求仕様書）の更新 (完了)**:
   * 「サンキューハザード」の発生条件（後方距離30m以下、8秒後の自動展開等のシーケンス）を具体的な要求として明記。
   * 「実行エンジン要求」におけるイグニッション連動とオーバーライドの原則を明記。
3. **SW205（アーキテクチャ設計書）の更新 (完了)**:
   * 「Vehicle State」の内部拡張キー構造を明記。
   * 「クルマシミュレータ」のエアコンUIなどの仕様を追記。
   * 「シナリオ・ビューア」の具体的なRead-Onlyロジックの詳細を追記。
   * 「シナリオ実行エンジン」の `UserMemoryState` に関する仕組みを追記。

## 3. 確認方法 (Verify)
* 対象ファイル（`.agent/docs/SW105_ソフトウェア要求仕様書.md` および `.agent/docs/SW205_ソフトウェアアーキテクチャ設計書.md`）内に、「前回」等の他文書依存の言葉が含まれていないことを目視および検索で確認する。
* ASDoQ文書品質モデル（目的適合性、完全性）を満たしているかマサにレビューを依頼する。
