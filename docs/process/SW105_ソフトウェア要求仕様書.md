# (SW105) ソフトウェア要求仕様書

## 2. ドキュメント構成（目次と記載要領）

[cite_start]以下は、ソフトウェア要求仕様書に記載すべき項目と、その記載内容のガイドラインです [cite: 2140, 2141, 2143, 2145, 2147, 2149, 2151, 2153, 2155]。

### 1. 概要
* [cite_start]**記載内容:** ドキュメントの目的、位置づけ、記載内容などの本書の概要および参照しているドキュメント名等を記載する [cite: 2193]。
* [cite_start]**記載例:** 本書の目的、本書の位置づけ、対象ユーザ、記載範囲・記載内容など、参照しているドキュメントなど、定義(用語、略語など) [cite: 2164, 2165, 2166, 2167, 2168, 2169]。

### 2. システム構成
* [cite_start]**記載内容:** ハードウェアを含めたシステム全体の構成とソフトウェアの位置づけ、およびソフトウェアを取り巻く関係/条件を記載する。システム要求仕様書やハードウェア仕様書等の関連資料より要求、条件などを整理して記載する [cite: 2194]。
* [cite_start]**記載例:** システム全体構成（システム構成要素の名称/基本機能、ソフトウェアに求められる要求仕様） [cite: 2172, 2173, 2174]。

### 3. 機能概要
* [cite_start]**記載内容:** 機能概要を箇条書きでまとめる。詳細は6項に記載する [cite: 2195]。
* [cite_start]**記載例:** システムとして実現・提供する機能のうち、ソフトウェアで実現する機能のリストと概要 [cite: 2178]。

### 4. 制約条件
* [cite_start]**記載内容:** 制約となる条件を漏れなく記載する [cite: 2196]。
* [cite_start]**記載例:** ハードウェア構成とその制約、利用するOS、ミドルウェアなどの制約 [cite: 2182, 2183]。

### 5. ユースケースとユースケースシナリオ
* [cite_start]**記載内容:** システムを構成する機能ブロックごとにユーザとソフトウェアとしてのやりとりを考慮し、時系列的にその流れを整理して記載する [cite: 2197]。
* [cite_start]**記載例:** 製品の利用シーンや利用コンテキストをもとにしたユースケースとユースケースシナリオ [cite: 2186]。

### 6. 機能詳細
* [cite_start]**記載内容:** 各機能の詳細を機能ごとに記載する [cite: 2198]。
* [cite_start]**記載例:** 5項で整理したユースケースを実現する機能の詳細 [cite: 2189]。

### 7. インタフェース詳細
* [cite_start]**記載内容:** インタフェース対象ごとインタフェースの詳細を記載する [cite: 2199]。
* [cite_start]**記載例:** 製品が連携動作する周辺のソフトウェア、システムやハードウェアなどとのインタフェース [cite: 2192]。

### 8. 性能・品質等非機能要求詳細
* [cite_start]**記載内容:** 機能で表現できない、性能・品質的な事項を記載する [cite: 2229]。
* **記載例:**
  * [cite_start]**信頼性要求:** システムの異常処理方式、システムの異常動作モードからの復帰手順や復帰方式 [cite: 2208, 2209, 2210]。
  * [cite_start]**使用性要求:** ソフトウェアで実現する部分の操作性、ハードウェアで実現する部分との接点 [cite: 2211, 2212, 2213]。
  * [cite_start]**効率性要求:** システムの実行性能(例:処理速度、起動時間、応答時間など)、リソース効率(例: メモリ容量、データサイズ) [cite: 2214, 2215, 2216]。
  * [cite_start]**保守性要求:** リモートメンテナンスなど保守の方式とその実現方法 [cite: 2217, 2218]。
  * [cite_start]**移植性要求:** ソフトウェアユニットの独立性 [cite: 2219, 2220]。
  * [cite_start]**セキュリティ要求:** 例:データ暗号化、ユーザ認証、ウィルス対策など [cite: 2221, 2222]。

### 9. その他
* [cite_start]**記載内容:** その他特記しておくべき事項がある場合には記載する [cite: 2230]。
* **記載例:**
  * [cite_start]セキュリティ要求(例: データ暗号化、ユーザ認証、ウィルス対策など) [cite: 2225]。
  * [cite_start]相互運用性(例: 通信プロトコルなど) [cite: 2226]。
  * [cite_start]外部インタフェース要求(例: 連携ソフトウェアとの関数インタフェース、通信プロトコル、ユーザインタフェースなど) [cite: 2227, 2228]。
