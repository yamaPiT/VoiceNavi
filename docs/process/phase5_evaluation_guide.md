# Phase 5 評価ガイド（推奨）

> [!NOTE]
> 本ガイドは **推奨ツール** です。Product Ownerはこの手順に従わなくても、自由な方法でプログラムを評価し、チャットやBUG101への記入だけでAIとの協働を進められます。
> 構造化された評価を行いたい場合にご活用ください。

---

## 1. 確認チェックリスト

プログラムの動作確認時に、以下の観点で評価することを推奨します。

### 基本動作の確認
- [ ] プログラムが正常に起動するか
- [ ] 主要な機能が要求仕様書（SW105）通りに動作するか
- [ ] UIの見た目・操作感が意図通りか

### 異常系の確認
- [ ] 不正な入力に対してエラーメッセージが表示されるか
- [ ] ネットワーク切断などの例外状況に対処できるか
- [ ] 長時間動作でメモリリーク等が発生しないか

### AIテスト結果の確認
- [ ] SWP6のテスト結果を確認し、全項目がPassしているか
- [ ] 手動テストが必要な項目（UI操作感等）を実施し、結果をSWP6に追記

---

## 2. バグ報告の書き方

バグを発見した場合は、`docs/artifact/BUG101_バグ管理表.md` に以下の情報を記入してください。

| 項目 | 記入例 |
| :--- | :--- |
| **発生機能・発生条件** | ログイン画面 / 空パスワード送信時 |
| **期待値と実際の結果** | 「パスワードを入力してください」と表示されるべきだが、画面がクラッシュする |

> **コツ**: 「再現手順」が具体的であればあるほど、AIは正確にバグを特定・修正できます。

---

## 3. 要求変更の判断ガイド

実際にプログラムを動かした結果、変更が必要と感じた場合の判断基準です。

| 変更の種類 | 例 | 対応方法 |
| :--- | :--- | :--- |
| **バグ修正** | 「ボタンを押しても反応しない」 | BUG101に記入 → AIがPhase 4でデバッグ |
| **仕様微調整** | 「ボタンの色を変えたい」 | チャットで指示 → AIがスケール判定して対応 |
| **新機能追加** | 「CSV出力機能を追加したい」 | チャットで指示 → Phase 1からループバック |
