## 1. はじめに

### 1.1 本書の目的
本書は、ムーブレボ通報システムを制御するためのソフトウェア要求を定義する。なお、ソフトウェアの名称は、「ムーブレボ通報システム・ソフトウェア」であり、以降、本書ではまぎれがない限り本ソフトウェアと呼び、ムーブレボ通報システムは通報システムと呼ぶ場合がある。

本書提供先と使用目的は、次のとおりである。
* **本ソフトウェアの発注者:** 本書により、本ソフトウェアの仕様をソフトウェア開発者と合意する。
* **ソフトウェアのアーキテクチャ設計者:** 本書で定義された要求に基づき、本ソフトウェアのアーキテクチャ設計を行う。
* **統合テスト設計者:** 本書で定義されたとおりにプログラムが作成されていることを確認するために、本ソフトウェアの統合テスト仕様を作成する。

## 2. ソフトウェア概要

### 2.1 本ソフトウェアの概要
本ソフトウェアは、[上位文書1]でその動作が定義されるムーブレボ通報システムを制御する。実現する機能概要を以下に示す。

**(1) 表示機能**
* 移動速度 (単位km/h, 最小単位1km/h, 最大表示30km/h)
* バッテリ残量 (単位%, 最小単位1%, 最大表示100%)
* 走行可能距離 (単位km, 最小単位1km, 最大表示100km)
* Status 表示として、通報システムの正常時には「正常」を、異常発生時には「異常」を点灯させる。

**(2) 通信機能**
異常の発生を検出した場合、通信システムに対して、異常発生の通知を行う。異常発生は以下のいずれかが成立することで検知する。
* 過度のG (3G以上)を検出
* ユーザが警報ボタンを押下

### 2.2 本ソフトウェアの動作環境
本ソフトウェアは、[上位文書2]で定義されるハードウェア(通報システムECU)に組込まれて動作する。
* **CPU:** XY32B/4L
* **外部インタフェースと接続するデバイス:**
  * CAN: 速度制御システム、バッテリ制御システム、通信システム
  * 入出力ポート: GPSセンサ、警報ボタン、LED、7セグ表示器
  * ADコンバータ: Gセンサ
* **RTOS:** μITRON4.0

### 2.3 外部入出力値の概要
*(※詳細は「3. 入出力値の定義」を参照)*

### 2.4 通報システムが取る状態
通報システムは以下の状態を遷移する。
* **非通電状態:** イグニッションスイッチがOFFの状態。
* **起動処理状態:** 本ソフトウェアが起動時の処理を実施している状態。
* **正常状態:** 起動時の処理が正常終了して、通常の制御を実施している状態。
* **異常状態:** 回復できない異常が発生し異常時の制御を実施している状態。

### 2.5 起動と終了
* **起動:** イグニッションスイッチがONになった時点で起動し、起動時自己診断を行う。
* **終了:** イグニッションがOFFになった時点、または電源電圧が下限値以下に低下した時点(SFR208)で非通電状態となり動作を終了する。

## 3. 入出力値の定義

### 3.1 G値 (入力)
* 入力先: Gセンサから入力し、AD変換#1で読み取る
* 入力タイミング: 100msec 周期
* 最小単位: 0.1G / 最小値: -3.5G / 最大値: 3.5G

### 3.2 GPS値 (入力)
* 入力先: GPSセンサから入力し、GPIO#1で読み取る
* 入力タイミング: 1sec 周期
* 緯度・経度 (北緯・東経のみ): 最小単位 0.1秒

### 3.3 警報ボタン値 (入力)
* 入力先: 警報ボタンの押下状態を、GPIO#2で読み取る
* 入力タイミング: 100msec 周期

### 3.4 移動速度値 (入力)
* 入力先: 速度制御システムから、CAN経由で読み取る
* 入力タイミング: 500msec 周期
* 最小単位: 1km/h / 最小値: 0km/h / 最大値: 30km/h

### 3.5 バッテリ残量値 (入力)
* 入力先: バッテリ制御システムから、CAN経由で読み取る
* 入力タイミング: 60sec 周期
* 最小単位: 1% / 最小値: 0% / 最大値: 100%

### 3.6 バッテリ電圧値 (入力)
* 入力先: バッテリ制御システムから、CAN経由で読み取る
* 入力タイミング: 60sec 周期
* 最小単位: 15/255v / 最小値: 0v / 最大値: 255v

### 3.7 走行可能距離値 (入力)
* 入力先: バッテリ制御システムから、CAN経由で読み取る
* 入力タイミング: 60sec 周期
* 最小単位: 1km / 最小値: 0km / 最大値: 50km

### 3.8 7セグ表示値 (出力)
速度、バッテリ残量、走行可能距離を表示するための値。

### 3.9 システム状態値 (出力)
システム状態値は'正常'と'異常'の2値を取る。

### 3.10 通報値 (出力)
G値超過または警報ボタン押下時に通信システムへ出力する値。

## 4. 開発制約要求

* **4.1 開発言語要求 (SFR101):** C言語 (C99)を用い、コーディングはMISRA-C:2004規格に準拠する。[規格文書3]に従う。
* **4.2 ミドルウェア要求 (SFR102):** RTOSは、TOPPERS/ASP (v1.07)を使用する。
* **4.3 設計制約 (SFR103):** [規格文書2]に準拠する。
* **4.4 開発環境要求 (SFR104):** OSはWindows7、コンパイラ群はGCCを使用する。
* **4.5 適用プロセス要求 (SFR105):** [規格文書1]に準拠する。

## 5. 機能要求

### 5.1 起動処理 (SFR201)
起動時にCPU/RTOSの初期化、ハードウェア診断、他システムの状態確認を行う。全て正常で正常状態へ、異常があれば異常状態へ遷移する。

### 5.2 移動速度表示機能 (SFR202)
* **正常状態:** 最新5回分の平均値を算出し、速度表示部に表示する（表示更新周期: 500msec）。
* **異常状態:** 'EE'と表示する。

### 5.3 バッテリ残量表示機能 (SFR203)
* **正常状態:** 60sec周期で入力した値を加工せずに表示する。
* **異常状態:** 'EE'と表示する。

### 5.4 走行可能距離表示機能 (SFR204)
* **正常状態:** 60sec周期で入力した値を加工せずに表示する。
* **異常状態:** 'EE'と表示する。

### 5.5 システム状態表示機能 (SFR205)
通信システムからACKが返ってこない等の場合に'異常'とする。正常時は「正常」LED点灯、異常時は「異常」LEDを点灯する。

### 5.6 G値超過通報機能 (SFR206)
正常状態において、G値が3G以上または-3G以下になった時点でCAN通信で送信する。

### 5.7 警報ボタン押下時通報機能 (SFR207)
正常状態において、警報ボタンが押下された状態に変化した時点でCAN通信で送信する。

### 5.8 電圧低下時終了機能 (SFR208)
電源電圧が動作保証電圧の下限値以下に低下した時点でプログラムの動作を終了する。

## 7. 保守・移植性要求

### 7.1 設計要求 (SFR401)
* **(1) ハードウェア依存部を独立させる (SFR401-1):** センサとアクチュエータのハードウェア依存部をモジュール化する。
* **(2) OS変更への準備 (SFR401-2):** 将来のAUTOSAR OS 4.0への移植を考慮し、対応するOSサービスのみを使用する。

### 7.2 コーディング要求 (SFR402)
* **(1) 英語コメントの使用 (SFR402-1):** プログラムのコメントは全て英語で記述する。
* **(2) コーディング規則 (SFR402-2):** MISRA-C:2004 に準拠する。

### 7.3 開発文書管理要求 (SFR403)
* **(1) 電子化 (SFR403-1):** 開発文書は、Office2010 の Word および Excel 形式で作成する。

## 8. 信頼性要求

### 8.1 テスト件数要求 (SFR501)
テスト件数は、コメント行を含むプログラムコード 1,000行あたり、100件以上を確保する。

### 8.2 コードカバレッジ要求 (SFR502)
テスト網羅度は、Modified condition/decision coverage (MC/DC) とする。
