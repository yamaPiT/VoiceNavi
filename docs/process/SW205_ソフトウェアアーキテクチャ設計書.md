# (SW205) ソフトウェア・アーキテクチャ設計書

【エージェントへの指示】
本ドキュメントは、SWP2（アーキテクチャ設計）の成果物である。SW105（ソフトウェア要求仕様書）で定義された要件を満たすための、ソフトウェアの「静的構造」と「動的振る舞い」を定義し、後工程（詳細設計および実装）の入力として曖昧さのない状態にすること。

## 1. 概要
本設計書全体の前提事項を定義する。
* **1.1 目的と位置づけ**: 本ドキュメントの目的、およびプロジェクトにおける役割を記述する。
* **1.2 適用範囲**: 本アーキテクチャ設計が対象とするソフトウェアの範囲を明確にする。
* **1.3 参照ドキュメント**: 設計の根拠となった資料（SW105等）やハードウェア仕様書を列挙する。
* **1.4 用語定義**: 設計において用いる固有のドメイン用語や略語を定義する。

## 2. システム構成
ハードウェアを含めたシステム全体におけるソフトウェアの立ち位置を定義する。
* **2.1 システム全体構成**: ソフトウェアが稼働するハードウェアデバイスや、連携する外部システムとの境界を記述する（SW105の情報をベースに具体化）。
* **2.2 主たるソフトウェア要素**: システム内で本ソフトウェアが担う主要な役割と構成要素を定義する。

## 3. ソフトウェア構成（静的構造）
ソフトウェア内部のモジュール分割と依存関係を定義する。
* **3.1 ソフトウェア全体構成**: ソフトウェア全体をどのように階層化・構造化するか（レイヤ構造、MVCなど）の基本方針を記述する。
* **3.2 機能ユニットの定義**: ソフトウェアを構成する各モジュール（機能ユニット）の名称、役割、および各ユニット間の関係性（依存関係）を明確にする。

## 4. 制御方式（動的振る舞いとリソース割り当て）
システム全体を正しく稼働させるためのメカニズムを定義する。
* **4.1 メモリ構成とレイアウト**: メモリマップ、ROM/RAMの使用計画、動的メモリ（ヒープ）やスタックの割り当て方針を記述する。
* **4.2 ソフトウェア制御方式**: タスク分割、優先順位、排他制御、プロセス間通信などの並行処理方式や、起動・停止シーケンス、状態遷移の方針を記述する。
* **4.3 性能見積り**: リアルタイム性などの非機能要件を満たすための、スループットや最大処理時間などの定量的な見積もり結果を記述する。

## 5. 機能ユニット詳細
* 3章で分割した各機能ユニット（モジュール）について、外部に公開するインタフェース（API、引数、戻り値）を定義し、後工程で実装可能なレベルまでブレイクダウンする。

## 6. システムで扱うデータ
* ソフトウェア全体で共有する主要なデータ構造（構造体、クラス）、グローバル変数、またはデータベースのスキーマなどを定義する。

## 7. 例外・異常処理一覧
* ハードウェアエラー、通信エラー、タイムアウトなどの異常ケースを網羅し、それぞれに対する検出方法とリカバリ手順（フェールセーフ動作など）を定義する。

## 8. その他・特記事項
* 現時点で未決定の事項（TBD）や、詳細設計以降へ引き継ぐべき注意事項などを記載する。
