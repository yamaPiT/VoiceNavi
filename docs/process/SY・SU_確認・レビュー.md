# システム・安全性テスト仕様書・報告書 および 共同レビュー記録

【エージェントへの指示】
本ドキュメントは、システム結合テスト（SYP3）、システムテスト（SYP4）、および安全性テスト（SAP2）における「テスト計画」「実行結果の記録」、および最終的な「ステークホルダ（人間）との共同レビュー記録（SU801相当）」を統合管理するためのものである。
QA_Agent等がテストを計画・実行（1〜3章）し、最後にReviewer_Agentが人間を交えたレビュー結果（4章）を記録すること。

---

## 【計画・実行フェーズ: QA_Agent / System_Integration_Agentが記載】

## 1. テスト概要と環境
* **1.1 テスト種別**: 本テストがシステム結合、システムテスト、安全性テストのいずれに該当するかを明記する。
* **1.2 参照ドキュメント**: システムアーキテクチャ設計書、システム要求仕様書、安全要求仕様書など、テストの根拠となるインプット情報を列挙する。
* **1.3 システム構成とテスト環境**: ソフトウェア単体ではなく、ターゲットハードウェアや外部システムを含めた全体構成、およびテストに使用する設備・ツールを定義する。

## 2. テスト方針と合否判定基準
* **2.1 テスト方針**: システム全体として満たすべき品質基準、および安全性（異常時のフェールセーフ動作等）に関する検証アプローチを定義する。
* **2.2 バグ収束の評価方針**: 不具合の発生状況（バグカーブ等）をどのように評価し、テスト完了の判断材料とするかを記述する。
