# テスト仕様書 / テスト報告書 (システム・安全性テスト用)

テスト仕様書には、テスト項目、テストデータなどを記載します。
テスト報告書には、テスト結果の評価（とらえられた問題と対応、問い合わせ件数とその回答、不具合件数、ソフトウェア変更有無の必要性など）を記載します。

## 1. プロセスにおける位置づけ

### 1.1 本ドキュメントを作成・利用する主なタスク
* **SYP3 システム結合テスト:** 3.1 準備、3.2 実施、3.3 結果の確認
* **SYP4 システムテスト:** 4.1 準備、4.2 実施、4.3 結果の確認、4.4 開発の完了確認
* **SAP2 安全性テスト:** 2.1 準備、2.2 実施、2.3 結果の確認

### 1.2 まとめる際に参照する情報 (インプット)
* **システム結合テスト:** (SY205) システム・アーキテクチャ設計書、(SY305) システム結合テスト結果、(SU601)不具合管理票
* **システムテスト:** (SY106) システム要求仕様書、製品マニュアル、(SY405) システムテスト結果
* **安全性テスト:** (SA104) 安全要求仕様書、(SA204) 安全性テスト結果

## 2. ドキュメント構成（目次と記載要領）

### 1. 概要
* **記載内容:** ドキュメントの目的、位置づけ、対象ユーザ、記載範囲・記載内容など、参照しているドキュメント名等を記載する。

### 2. テスト概要
* **2.1 テスト方針:** テストに対する基本的な考え方を記載する。
* **2.2 開発フェーズおよびテスト期間:** テストの対象となるソフトウェアの開発フェーズ、ソフトウェア/ハードウェアのバージョンおよびテスト期間を記載する。
* **2.3 対象範囲:** 本仕様書にて実施されるテスト対象範囲 (機能)を記載し、関連する仕様書やアーキテクチャ設計書とテスト項目との対応づけをする。

### 3. 評価設備・環境
* **3.1 システム構成:** システム構成とテスト対象の(ソフトウェア)関係を図等を使用し分かりやすく記載する。
* **3.2 テスト環境・設備:** テスト実施時に使用するテスト環境（設備、メーカ、型番、シリアル番号、台数、用途）を図等を使用して分かりやすく記載する。ツールとの接続方法も含む。

### 4. テスト概要/結果概要
* **記載内容:** テスト内容と期待する結果を記載する。分類ID、分類名、テスト項目数、検査開始日/終了日、不具合数、未対応数、備考などをサマリとして一覧表で記載する。

### 5. 添付資料
* **5.1 テスト項目詳細:** テスト項目IDごとに、入力データ/環境条件、テスト実施手順(設備・ツール使用方法と注意点)、期待出力、検査結果、完了日などを記載する。
* **5.2 バグカーブ:** テスト工程におけるバグ収束状況を確認するために、縦軸にバグ累積数、横軸に時間をとってグラフ等で記載する。
* **5.3 その他:** テスト項目の分類基準(正常値/異常値入力、過去不具合、負荷テストなど)、テスト結果判定基準(OK、NGなど)、テスト結果を客観的に証明する添付資料(波形データなど)を記載する。
