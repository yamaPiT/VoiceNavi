# (SW601) ソフトウェア総合テスト仕様書 / (SW606) ソフトウェア総合テスト報告書

## 2. ドキュメント構成（目次と記載要領）

以下は、ソフトウェア総合テスト仕様書および報告書に記載すべき項目と、その記載内容のガイドラインです。

### 1. 概要
* [cite_start]**記載内容:** ドキュメントの目的、位置づけ、記載内容などの本書の概要および参照しているドキュメント名等を記載する。[cite: 2177]
* [cite_start]**記載例:** 本書の目的、本書の位置づけ、対象ユーザ、記載範囲・記載内容など、参照しているドキュメントなど、定義(用語、略語など)。[cite: 2149, 2150, 2151, 2152, 2153, 2154]

### 2. テスト概要
#### 2.1 テスト方針
* [cite_start]**記載内容:** テストに対する基本的な考え方を記載する。[cite: 2178]
#### 2.2 開発フェーズおよびテスト期間
* [cite_start]**記載内容:** テストの対象となるソフトウェアの開発フェーズ、ソフトウェアバージョン、ハードウェアバージョンおよびテスト期間を記載する。[cite: 2179]
#### 2.3 対象範囲
* [cite_start]**記載内容:** 本仕様書にて実施されるソフトウェアのテスト対象範囲 (機能)を記載する。[cite: 2180] [cite_start]ソフトウェア要求仕様書やソフトウェア・アーキテクチャ設計書とテスト項目との対応づけをする。[cite: 2180]

### 3. 評価設備・環境
#### 3.1 システム構成
* [cite_start]**記載内容:** システム構成とテスト対象の(ソフトウェア)関係を図等を使用し分かりやすく記載する。[cite: 2201]
#### 3.2 テスト環境・設備
* [cite_start]**記載内容:** テスト実施時に使用するテスト環境(ハードウェア環境など)を図等を使用して分かりやすく記載する。(テストにて使用するツールとの接続方法も含む)。[cite: 2202, 2203] [cite_start]実施時に使用する設備の情報(バージョン情報など)と目的を明記する。[cite: 2204]
* [cite_start]**記載項目例:** 設備、メーカ、型番、シリアル番号、台数、用途 [cite: 2189]

### 4. テスト概要/結果概要
* [cite_start]**記載内容:** テスト内容と期待する結果を記載する。[cite: 2205] [cite_start]各テスト項目には管理しやすいように番号を振り、テストの実施日やテスト結果をサマリとして記載する。[cite: 2205]
* [cite_start]**記載項目例:** 分類ID、分類名、テスト項目数、検査開始日/終了日、不具合数、未対応数、備考 [cite: 2192]

### 5. 添付資料
#### 5.1 テスト項目詳細
* [cite_start]**記載内容:** テスト項目ごとにテストデータ、期待する出力、結果などを記載する。[cite: 2206, 2207]
* [cite_start]**記載項目例:** テスト項目ID、入力データ/環境条件、テスト実施手順(設備・ツール使用方法と注意点)、期待出力、検査結果、完了日。[cite: 2230] [cite_start]※テスト未実施項目とその理由なども必要に応じて記載する。[cite: 2230]
#### 5.2 バグカーブ
* [cite_start]**記載内容:** テスト工程におけるバグ収束状況を確認するために、縦軸にバグ累積数、横軸に時間をとってその関係をグラフ等で記載する。[cite: 2208, 2209, 2210, 2211]
#### 5.3 その他
* [cite_start]**記載内容:** ソフトウェア全体を通して、開発グループ全員で情報共有すべき事柄やテスト証跡などを記載する。[cite: 2222, 2223]
* [cite_start]**記載例:** テスト項目の分類基準(正常値入力、異常値入力、過去不具合、負荷テストなど)、テスト結果判定基準(OK、NGなど)、テスト結果を客観的に証明する添付資料(波形データなど)。[cite: 2215, 2216, 2217, 2218]
