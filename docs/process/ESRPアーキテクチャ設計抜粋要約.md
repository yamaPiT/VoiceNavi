# エージェント主導型 ソフトウェア・アーキテクチャ設計ガイドライン

本ガイドラインは、AIエージェントが組込みソフトウェアのアーキテクチャ設計（SWP2）を自律的に行い、妥当性を検証するための標準タスクを定義する。

## 1. 開発プロセスの全体像（SEP体系）
エージェントは、以下のソフトウェア・エンジニアリング・プロセス（SEP）を前提に作業を行う。
* **SWP1**: ソフトウェア要求定義（出力: SW105 ソフトウェア要求仕様書）
* **SWP2**: ソフトウェア・アーキテクチャ設計（★本ガイドラインの対象 / 出力: SW205）
* **SWP3**: ソフトウェア詳細設計
* **SWP4**: ソフトウェア・コード作成とテスト
* **SWP5**: ソフトウェア結合
* **SWP6**: ソフトウェア適格性確認テスト（総合テスト）

---

## 2. SWP2: ソフトウェア・アーキテクチャ設計の実行タスク（Architect_Agent向け）

エージェントは、以下のプロセスに従ってアーキテクチャ設計書を構築すること。

### SWP2.1.1 設計条件の確認（インプット分析）
* **入力ファイル**: `SW105_ソフトウェア要求仕様書`、および関連するハードウェア仕様書
* **タスク**: SW105を読み込み、実現すべき機能要件・非機能要件、およびハードウェア/ソフトウェアの制約（開発言語、OSなど）を特定して設計の境界線を確定する。

### SWP2.1.2 ソフトウェア静的構造の設計
* **タスク**: システム全体を機能ユニット（モジュール）に分割する。共通処理の抽出、変更容易性を考慮した機能の局所化を行い、各モジュールの位置関係（SW201相当）を定義する。

### SWP2.1.3 ソフトウェア動的振る舞いの設計
* **タスク**: 並行処理、タスクの優先順位、排他制御を明確にし、異常時のリカバリ動作を含む状態遷移や動作シナリオ（SW202、SW203相当）を定義する。

### SWP2.1.4 / SWP2.1.5 インタフェースおよびリソースの設計
* **タスク**: モジュール間のデータ受け渡し手順（API定義等：SW204相当）を決定し、メモリ空間のレイアウトや使用量のピーク値、処理性能の試算を行う。

### SWP2.1.6 ソフトウェア・アーキテクチャ設計書の作成と検証（Reviewer_Agent向け）
* **出力ファイル**: `SW205_ソフトウェアアーキテクチャ設計書`
* **タスク**: これまでの設計事項（SW201〜SW204相当）を整理し、SW205として文書化する。出力前に以下の内部検証を実施すること。
  * **要件の網羅性**: SW105に記載されたすべての要件がSW205にマッピングされているか。
  * **実現可能性**: 見積もったリソース（メモリ・性能）で設計構造が破綻していないか。
