# ASDoQ システム開発文書品質 チェックリスト（軽量版）

> 本ファイルは `asdoq_model_markdown.md`（フル版）から、6大品質特性 × 副特性 × 測定項目のみを抽出した軽量チェックリストです。  
> **通常のレビュー作業ではこの軽量版を使用**してください。  
> 詳細な例文・違反例が必要な場合は、Product Ownerに「ASDoQ文書品質モデルに基づき厳密にレビューして」と指示し、フル版 `asdoq_model_markdown.md` をロードしてください。

---

## 1. 目的適合性
*記述内容が、目的達成のために必要十分であるかの度合い*

### 1.1 目的明示性
- [ ] 文書の読み手（属性・特性）を記述している
- [ ] 文書の読み手の範囲（提出先・配布先・開示範囲）を記述している
- [ ] 文書の適用範囲を記述している
- [ ] 文書の目的を記述している
- [ ] 文書の目的を達成するために必要な項目を記述している

### 1.2 目的合致性
- [ ] 明示した目的を達成するために必要な内容を記述している

---

## 2. 保守性
*記述内容を利用維持できる度合い*

### 2.1 再利用性
- [ ] 記述内容を部品化できる
- [ ] 記述の基準または標準を定めている
- [ ] 用語集を作成している

### 2.2 持続可能性
- [ ] 現在および将来において、記述内容が実現可能である
- [ ] 記述内容が検証可能である
- [ ] 「未決定」事項を解消する決定時期・決定方法を記述している

---

## 3. 整合性
*記述内容を論理的に展開している度合い*

### 3.1 無矛盾性
- [ ] 項目間の矛盾がない
- [ ] 条件間の矛盾がない
- [ ] 上位と下位が対応している
- [ ] 記述間および文書間の関係が整合している

### 3.2 一貫性
- [ ] 原因と結果が対応している
- [ ] 理由または根拠を記述している
- [ ] 記述間の接続関係（展開・順序）が適切である
- [ ] 全体像を明示した後に詳細を記述している
- [ ] 記述単位の見出しと本文が合致している
- [ ] 図表と本文とに齟齬がない

### 3.3 構造性
- [ ] 文書の構成（見出しレベルの付け方など）が適切である
- [ ] 図表内の構成が整っている

---

## 4. 理解容易性
*記述内容を理解しやすく表現している度合い*

### 4.1 非曖昧性（一意性）
- [ ] 複数の解釈が成り立たない
- [ ] 1つの文内で1つの事柄のみを記述している
- [ ] 同じ内容のものを違う名前で定義しない
- [ ] 指し示すものが明確である
- [ ] 表に空欄を残さない
- [ ] 状態とイベント（事象）を区別している

### 4.2 非曖昧性（識別性）
- [ ] 動作を特定できる
- [ ] 状態を特定できる
- [ ] 対象を特定できる
- [ ] 違う内容のものを同じ名前で定義しない
- [ ] 意味を特定しやすい助詞を使用している

### 4.3 関係性
- [ ] 関連する複数箇所の記述に対して、参照情報を明示している
- [ ] 記述内容は追跡可能である

---

## 5. 可読性
*記述内容を読みやすく記載している度合い*

### 5.1 簡潔性
- [ ] 多重の入れ子文がない
- [ ] 記述に重複がない
- [ ] 二重否定を使用しない
- [ ] 不要な表記および表現がない

### 5.2 表記適切性
- [ ] 文体を統一している
- [ ] 用語および表記法を統一している
- [ ] 用語を使い分けている
- [ ] 用語（専門用語・略語など）および記号を定義している
- [ ] 用字用語が適切である
- [ ] 記号類・括弧類を適切に使用している

### 5.3 表現適切性
- [ ] 視点を統一している
- [ ] 行為の主体を特定できる
- [ ] 箇条書きまたは図表を適切に使用している
- [ ] インデントまたは改行を効果的に使用している
- [ ] 読みやすいレイアウトやフォント種を使用している

---

## 6. 規範適合性
*記述内容を標準や規則に準拠して記載している度合い*

### 6.1 制約適合性
- [ ] ドメインで扱うデータの定義域、処理可能容量などが適切である
- [ ] 開発制約として対象言語の書記体系に適合している

### 6.2 基準適合性
- [ ] 職務分掌に則った権限委譲などのビジネスルールに従っている
- [ ] 国内だけでなく、グローバルな環境にも適合している
- [ ] 技術領域の基準または標準に準拠している
- [ ] 文書標準に準拠している
- [ ] 事実と意見を書き分けている

### 6.3 文法適合性
- [ ] 文法的に正しい文である
- [ ] 接続詞の使用が正しい
- [ ] 助詞の使用が正しい
- [ ] ねじれ文がない
- [ ] 係り受けが正しい

### 6.4 記法適合性
- [ ] 箇条書きのルールに適合している
- [ ] 図表の記載ルールに適合している
- [ ] 各種の記法（状態遷移表・UMLなど）に則している
- [ ] 用字用語が正しい（誤字脱字がない）
