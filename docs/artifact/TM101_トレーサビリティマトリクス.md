# (TM101) トレーサビリティ・マトリクス

> [!NOTE]
> 本テンプレートは**推奨ツール**です。DADAプロセスの必須ステップではありませんが、要求・設計・テスト間の対応関係を可視化し、漏れや乖離を防ぐために活用を推奨します。

【運用ルール】
1. **要求定義（Phase 1）完了後**に、要求ID列を埋めます。
2. **設計（Phase 2）完了後**に、対応する設計ユニットID列を埋めます。
3. **テスト仕様（Phase 3）完了後**に、対応するテストID列を埋め、網羅性を確認します。
4. 各列が空欄のままの行があれば、**対応漏れ（未実装・未テスト）** の可能性があります。

## マトリクス

| 要求ID (SW105) | 要求概要 | 設計ユニットID (SW205) | テストID (SWP6) | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| REQ-001 | （例: ユーザー認証機能） | （例: AUTH-Module） | （例: TC-001, TC-002） | |
| REQ-002 | | | | |
| REQ-003 | | | | |

## 網羅性チェック

- [ ] すべての要求ID（SW105）に対応する設計ユニットID（SW205）が存在する
- [ ] すべての要求ID（SW105）に対応するテストID（SWP6）が存在する
- [ ] 設計ユニットIDに対応する要求が不明な項目がない（設計の独走がない）
