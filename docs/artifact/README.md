# docs/artifact/ ディレクトリについて

本ディレクトリは、DADAプロセスで作成される**開発成果物と管理帳票**を格納します。

## 格納ファイル

### 開発成果物（AIが作成、Product Ownerが承認）

| ファイル | 役割 | 作成フェーズ |
| :--- | :--- | :--- |
| `SW105_ソフトウェア要求仕様書.md` | 要求仕様の定義 | Phase 1 |
| `SW205_ソフトウェアアーキテクチャ設計書.md` | アーキテクチャ設計 | Phase 2 |
| `SWP6_ソフトウェア総合テスト仕様書・報告書.md` | テスト計画と結果 | Phase 3〜4 |

### 管理帳票（人間とAIが協働で記入）

| ファイル | 役割 | 備考 |
| :--- | :--- | :--- |
| `REV101_Requirements.md` | 要求定義レビュー記録表 | Phase 1 で使用 |
| `REV101_Design.md` | 設計レビュー記録表 | Phase 2 で使用 |
| `REV101_Test.md` | テストレビュー記録表 | Phase 3 で使用 |
| `BUG101_バグ管理表.md` | バグの報告・追跡・解決記録 | Phase 4〜5 で使用 |
| `TM101_トレーサビリティマトリクス.md` | 要求↔設計↔テストの対応表 | 推奨ツール |

## 運用ルール

1. **開発成果物**はAIが各Phaseで作成し、Product Ownerの承認を経て確定します。
2. **REV101（レビュー記録表）** は、人間が指摘を記入 → AIが修正して対応結果を追記、という非同期コラボレーションに使います。
3. **BUG101（バグ管理表）** は、人間がバグを記入 → AIが自律デバッグして原因と修正内容を追記します。
4. **TM101（トレーサビリティ・マトリクス）** は推奨ツールです。各Phase完了時に「TM101も更新して」とAIに指示することで活用できます。
