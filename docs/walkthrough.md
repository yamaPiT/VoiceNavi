## ユニットテスト生成 (`src/lib/store.ts`)

### 実施内容
マサの指示により、`@[/generate-unit-tests]` ワークフローを使用し、システムの状態管理を担うコアファイル `src/lib/store.ts` に対する包括的なユニットテストを自動生成しました。

### テスト環境の構築
プロジェクトにテストフレームワークが存在しなかったため、以下のモジュールを導入・構成しました。
* `vitest` (テストランナー)
* `@testing-library/react`, `@testing-library/jest-dom` (DOM/React拡張用)
* `jsdom` (テスト環境用)

### 実装内容 (`store.test.ts`)
Zustandストア（VehicleStore）の各種ロジックを検証する全14ケースのテストを実装しました。

1. **Initial State (初期状態)**:
   * Vehicle.IgnitionState の初期値確認 (`STOP`)
   * 各種アクチュエータの初期値確認
   * デフォルトの2シナリオ（Rain, Hazard）が登録されていることの確認
2. **Scenario Management (シナリオ管理)**:
   * `addScenario`: 新規追加の成功、重複IDのブロック
   * `updateScenario`: 既存シナリオの更新
   * `setScenarioRunning`: 実行時/停止時におけるキャッシュ(`PreRunStateCache`)やフラグ(`ManualOverrideFlags`)の初期化およびクリアロジック
3. **State Update Logic (VSS更新・依存ロジック)**:
   * 一般的な値の更新
   * IgnitionState が `STOP` になった際のアクチュエータ（ワイパー、デフロスタ、ファン等）一斉初期化
   * 雨量 (RainIntensity) の境界変動に伴う `WasRainBelow10` フラグの動作
   * IgnitionState が `START` になった際の `UserMemoryState` 初期化
   * `START` 中のアクチュエータ手動操作に伴う `ManualOverrideFlags` および `UserMemoryState` への記録
4. **User Interaction**:
   * プロパティ変更時に `lastUserInteract` へ直近の操作時刻が正確に記録されること

### 実行結果
`npx vitest run src/lib/store.test.ts` を実行し、**全14ケースがエラーなくPass（成功）** することを確認しました。これにより、複雑な条件分岐やキャッシュ制御を含むストアロジックの正確性が保証されました。
